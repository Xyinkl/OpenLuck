🔗 为什么用区块哈希？

以太坊每 ~12 秒产生一个新区块。每个区块的哈希值由全网数千个验证者共同决定， 任何人——包括本网站——都无法提前预测或篡改。 选取一个未来区块的哈希作为随机种子，结果完全公开、可独立验证。

📐 抽奖算法（可自行验证）
1
确定参与人数 N，每人分配编号 1…N
2
选定一个未来区块号 B，公告所有参与者
3
等待区块 B 被以太坊网络打包确认
4
从链上取得区块 B 的 256 位哈希 H（64 个十六进制字符）
5
迭代生成中奖号码：seed = SHA256(H + round)，取 seed mod N + 1，跳过重复，直到凑满中奖人数
6
任何人可用相同算法独立还原结果，无需信任本站

# ── 验证中奖结果的 Python 代码 ── import hashlib, requests BLOCK_NUMBER = 21000000 # 替换为实际区块号 TOTAL = 100 # 总参与人数 COUNT = 3 # 中奖人数 # 1. 获取区块哈希 r = requests.post("https://cloudflare-eth.com", json={ "jsonrpc": "2.0", "method": "eth_getBlockByNumber", "params": [hex(BLOCK_NUMBER), False], "id": 1 }) block_hash = r.json()["result"]["hash"][2:] # 去掉 0x 前缀 # 2. 迭代生成中奖号码 winners, selected, rnd = [], set(), 0 while len(winners) < COUNT: seed = f"{block_hash}{rnd:04x}".encode() h = hashlib.sha256(seed).hexdigest() num = int(h[:16], 16) % TOTAL + 1 if num not in selected: selected.add(num) winners.append(num) rnd += 1 print("区块哈希:", block_hash) print("中奖号码:", sorted(winners))
